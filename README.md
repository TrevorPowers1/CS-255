# CS-255

Briefly describe the DriverPass project, the client, and the type of system they wanted you to design.

The client for the project was DriverPass, an organization that assists students in getting ready for driving tests. The client wanted a web-based system that would permit students to schedule driving lessons, practice tests, and access DMV study materials all under the same umbrella. The objective of the assignment was to create a system that makes the learning process more organized for the students while also making the learning process more efficient for the instructors.

What did you do particularly well?

I believe that I did an excellent job of taking the client’s requirements and turning them into diagrams that present the client’s requirements in an organized way. Instead of simply describing the client’s requirements, I was able to create diagrams such as the use case diagram, activity diagram, sequence diagram, and class diagram, which illustrate the way the client’s requirements would work out in the real world. Additionally, I was able to keep the client’s requirements organized by focusing on the main services that the client cares about, such as the scheduling of lessons, the preparation of tests, etc.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise any part of the assignment, I would revise the activity diagram by including more alternate paths, such as the way the program would work if the time slot is not available or the way the program would work if the user exits the test before the completion of the test. This would make the program more complete, as the user may not always complete the test as intended.

How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?

I understood what the user needed based on my analysis of the interview from DriverPass. I concentrated on what the company really needed: a system where students could schedule their lessons and get ready for the DMV test. I based my system design on what the user needed by ensuring the diagrams focused on the students’ needs. The importance of taking into account the user’s needs is ensuring the system is effective in solving their issues. If the system does not meet the user’s needs, it is just a system for the sake of a system. 

How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?

I approach software design by first ensuring I have an idea of what the user wants and what the system is meant to achieve before concentrating on the technical details. I also prefer breaking it down into smaller steps using diagrams so I can get a better idea of how everything works together. In the future, I would use requirement analysis and modeling diagrams as a technique for designing a system more effectively. I would also ensure I get feedback on the system as it is still in the designing stage, as this is helpful in ensuring the system is more effective for the users.
